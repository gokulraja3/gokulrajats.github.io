Introduction

Jetpack has introduced this library to improve the startup time of the application.Many libraries run initializing code using Content Providers (eg: WorkManager, Clevertap) before Application.onCreate(). Apps having multiple content-providers slows the startup time and results in bad experience for users.

Jetpack App-Startup comes with excellent solution to solve this initialization cost.

App Startup Library

Previously we were using separate Content Providers for each library adding up a huge startup cost. App Startup Library uses Single Content Provider improving app startup time.

Setup

To use Jetpack Startup in your library or app, add the following to your Gradle file: 

implementation "androidx.startup:startup-runtime:1.0.0"

Implementing Component Initializers

  • We should define each component, a initializer by creating a class that implements Initializer< T >
  • Node.js v12 or higher
